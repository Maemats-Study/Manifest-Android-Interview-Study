# 장기적으로 실행되는 백그라운드 작업을 어떻게 관리하나요?

## 1. 조건부 작업에 적합한 WorkManager

- 앱이 닫히거나 기기 재부팅 후에도 실행되어야 하는 작업의 경우
- 백그라운드 작업을 관리하고 네트워크 가용성 또는 충전 상태와 같은 제약 조건 하에서 작업이 실행되도록 보장
- ex. 로그 업로드, 데이터 동기화, 비디오와 같이 영상 파일 업로드

## 2. 오랜 작업에 적합한 Service

- 음악 재생이나 위치 추적과 같이 지속적이고 오랜 실행이 필요한 작업
- 서비스는 UI와 독립적으로 실행되며 앱이 백그라운드에 있을 때도 계속 실행될 수 있음
- 작업이 노티피케이션과 함께 사용자가 인지할 수 있는 상황에서 실행되어야 하는 경우 Foreground Services 사용

## 3. Kotlin Coroutines 및 Dispatchers

- 앱 생명주기에 연결딘 작업의 경우
- 무거운 작업을 오프로드하려면 Dispatchers.IO를 사용하고 CPU 집약적인 계산에는 Dispatchers.Default를 사용
- 해당 접근 방식은 앱이 닫힌 후에도 유지될 필요가 없는 작업에 이상적

## 4. 시스템 수준 작업에 적합한 JobScheduler

- 작업이 기기 전체 작업과 관련되고 특정 조건(가령, 충전 중에만 실행)이 필요한 경우
- 즉시 실행이 필요하지 않은 작업에 적합
- WorkManager는 어차피 내부적으로 JobScheduler 등을 활용 (일반적인 상황에서는 WorkManager 권장)
