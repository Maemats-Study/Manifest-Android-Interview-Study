# Jetpack Paging 라이브러리

- 대규모 데이터 셋을 청크 또는 페이지 단위로 로드하고 표시하는 프로세스를 돕도록 설계된 안드로이드 아키텍처 컴포넌트
- 데이터베이스나 API와 같은 소스에서 데이터를 효율적으로 가져와야 하는 애플리케이션에 특히 유용
- 메모리 사용량을 최소화하고 RecyclerView 기반 UI의 전반적인 성능을 향상시킴
- Paging 라이브러리는 데이터를 점진적으로 로드하기 위한 구조화된 접근 방식을 제공
- 데이터 캐싱, 재시도 메커니즘, 효율적인 메모리 사용과 같은 주요 측면을 기본적으로 처리
- 해당 라이브러리는 로컬 데이터 소스와 원격 소스 또는 이 둘의 조합을 모두 지원

## Paging 라이브러리의 구성 요소

1. PagingData
    - 점진적으로 로드되는 데이터 스트림을 나타냄
    - Recyclerview와 같은 UI 컴포넌트에 의해 관찰되고 사용될 수 있음
2. PagingSource
    - 데이터 소스에서 데이터가 로드되는 방식을 정의하는 역할
    - 위치 또는 ID와 같은 키값을 기반으로 데이터 페이지를 로드하는 메서드를 제공
3. Pager
    - PagingSource와 PagingData 간의 중개자 역할을 함
    - PagingData 스트림의 생명주기를 관찰
4. RemoteMediator
    - 로컬 캐싱과 원격 API 데이터를 결합할 때 경계 조건을 구현하는 데 사용

## Paging 라이브러리 작동 방식

- 데이터를 페이지로 분할하여 효율적인 데이터 로딩을 가능하게 함
- 사용자가 RecyclerView를 스크롤하면 라이브러리는 필요에 따라 새 데이터 페이지를 가져와 최소한의 메모리 사용량을 보장
- 이 라이브러리는 Flow 또는 LiveData를 기본적으로 지원 → 데이터 변경 사항을 관찰하고 그에 따라 UI를 업데이트할 수 있음

### 일반적인 워크플로우

1. PagingSource를 정의하여 데이터 가져오는 방법을 지정
2. Pager를 사용하여 PagingData의 Flow를 생성
3. ViewModel에서 PagingData를 관찰하고 RecyclerView에서 렌더링하기 위해 PagingDataAdapter에 전달
