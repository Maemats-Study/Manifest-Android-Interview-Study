# Q) 11. 상태(State)란 무엇이며 이를 관리하는 데 사용되는 API는 무엇인가요?

Jectpack Compose 에서 상태를 관리하기 위해 여러 API 를 제공한다.

### Compose 에서 상태 관리

1. `remember`: 초기 컴포지션 발생시 메모리에 객체를 저장하고 recomposition 이 발생하면 기존 메모리에 저장된 값을 꺼내옵니다.
2. `rememberSavable`: recomposition 뿐 아니라, 화면 회전과 같은 구성 변경 시에도 화면 상태를 유지합니다. Bundle 에 저장할 수 있는 유형(`parcelable`) 또는 그 외 유형에 대해서는 커스텀 saver 객체와 함께 작동합니다.
3. `mutableStateOf`: 가변적인 상태를 의미, 상태 값이 변경될 때 recomposition 을 트리거하는 관찰 가능한 상태 객체를 생성

### 실전 질문

Q) 상태는 recomposition과 어떤 관련이 있으며, recomposition이 발생하면 내부적으로 어떤 일이 발생하나요?

`mutableStateOf` 로 설정된 값과 같은 상태는 변경시에 `recomposition` 을 트리거하여 데이터의 변경 사항(상태)을 UI에 반영합니다. 다만 단순 `mutableStateOf` 를 사용시 리컴포지션 되면서 해당 값을 잃기 때문에 해당 값을 잃지 않기 위해 `remember` 로 메모리에 값을 저장 하고, 리컴포지션시에 해당 메모리에 값을 다시 불러오게 합니다.
