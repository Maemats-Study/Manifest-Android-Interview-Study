# Q) 51. ViewBinding를 사용하면 어떤 장점이 있나요?

ViewBinding 은 레이아웃의 뷰와 상호 작용하는 프로세스를 단순화하기위해 안드로이드에서 도입된 기능

기존의 방식은 `findViewById` 를 사용한 방식으로 뷰에 접근하는 타입-세이프(type-safe) 방식을 제공해
보일러 플레이트 코드를 줄이고 잠재적 런타임 오류를 최소화합니다.

### ViewBinding 작동 방식
1. ViewBinding을 활성화하면 안드로이드는 각 XML 레이아웃 파일에 대한 바인딩 클래스 생성<br/>
생선된 바인딩 클래스의 이름은 레이아웃 파일의 이름에서 `카멜케이스Binding` 과 같이 생성됩니다. (activity_main.xml -> ActivityMainBinding)
2. 바인딩 클래스에는 레이아웃의 모든 뷰에 대한 참조를 가지고 있어 이를 캐스팅 하는 등의 수고가 없습니다.

### ViewBinding의 장점
- 타입 안전성(Type Safety):<br/>
캐스팅할 필요 없이 뷰에 직접 접근 하여 타입 불일치로 인한 런타인 오류 제거
- 보일러 플레이트 코드 제거
- Null 안전성:<br/>
nullable 타입의 뷰를 자동으로 처리하여 선택적 UI 컴포넌트와 상호 작용할 때 더 안전한 코드 보장
- 성능:<br/>
DataBinding 과 달리 ViewBinding 은 바인딩 표현식이나 추가 XML 파싱을 사용하지 않아 런타임 오버헤드가 최소화<br/>
데이터를 뷰에 직접 결합하지 않는 경우에는 ViewBinding 을 쓰는게 런타임 오버헤드가 줄어 더 가볍기 때문에 좋습니다.
