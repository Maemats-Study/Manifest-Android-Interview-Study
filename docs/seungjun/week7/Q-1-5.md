# Q) 5. Jetpack Compose의 안정성(stability)이란 무엇이며, 성능과 어떤 관련이 있나요?

Jetpack Compose 의 안정성(Stability) 는 클래스나 타입이 동일한 매개변수 입력값에 대해 일관된 결과를 생성하도록 보장하는 속성을 나타냅니다.

### Stable vs Unstable

**stable**

- 원시타입을 읽기 전용인 val 로 정의:

  String 을 포함한 원시 타입(Primitive types)  된다면, 고유한 값은 변경되지 않으므로 stable

- 값을 캡처하지 않는 람다:

  같은 함수 타입도 동작이 예측 가능하기에 stable 로 간주

- 읽기 전용이거나 변경 불가능하고 (immutable) stable 한 클래스( @Stable 또는 @Immutable 붙여있는 경우)

  public 속성을 가진 데이터 클래스는 stable, 같은 안정성 어노테이션 명시적으로 붙어있는 클래스는 stable 로 간주


**Unstable**

- 인터페이스 및 추상 클래스

  List, Map 또는 Any와 같은 인터페이스는 컴파일 시점에 구현체의 종류를 보장할 수 없기 때문에 반드시 unstable로 간주

- 가변적인 프로퍼티를 가진 클래스

  하나 이상의 가변적이거나 본질적으로 변경 가능하다고 간주되는 타입의 public 프로퍼티를 단 하나라도 포함하는 클래스는 unstable로 분류
